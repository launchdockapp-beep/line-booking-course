### 🧰 設定關卡 1（老師，課程開始）— 建 Firebase（四件事，缺一不可）

> ⭐ **最常漏掉的觀念**：「建立 Web 應用程式」（第 4 步，拿 `firebaseConfig`）和「建立資料庫」（第 2 步）是**兩件完全不同的事，兩個都要做**。很多人只做了 Web 應用程式就以為完成，結果根本沒有資料庫，連線必定失敗。

**① 建立專案**
1. 開 https://console.firebase.google.com ，用教室專用 Google 帳號登入。
2. 點 **「建立專案 / Create a project」** → 取專案名（例 `line-booking-course`）→（可關閉 Google Analytics）→ 建立 → 跑完點「繼續」。

**② 建立 Firestore 資料庫（← 最容易漏的一步，沒做後面全卡）**
1. 左側選單找 **「Firestore Database」**（橘色疊卡圖示；在「建構 / Build」或「專案捷徑」下）。
2. 進去後，主畫面中央點 **「建立資料庫 / Create database」**。
3. 模式：選 **「正式版模式 / Production mode」**（**不要**選測試模式）→ 下一步。
4. 位置：選 **`asia-east1`（台灣彰化）** → 「啟用 / 建立」。
5. 等約 30 秒～1 分鐘。看到空白的「資料」畫面、**上方出現「資料 / 規則 / 索引」分頁** = 資料庫建好了。

**③ 啟用登入方式**
1. 左側 **Authentication** → 「開始使用 / Get started」→ 點上方 **「Sign-in method」** 分頁。
2. 啟用 **「匿名 / Anonymous」**（點它 → 開啟 → 儲存）。
3. 啟用 **「Google」**（點它 → 開啟 → 選一個支援信箱 → 儲存）。

**④ 建立 Web 應用程式（拿 firebaseConfig）**
1. 左上 **齒輪 → 專案設定 / Project settings** → 往下捲到「你的應用程式」。
2. 點 **`</>`（Web）** 圖示 → 取暱稱 → **不要勾** Firebase Hosting → 「註冊應用程式」。
3. 複製畫面上的 `firebaseConfig` 物件（只要那串值，等下貼進前端）。

### 🧰 設定關卡 2（老師，課中）— 貼 config＋授權網域＋規則
1. 把 `firebaseConfig` 值貼進 firebase-init.js 的常數區（叫 agy 幫貼也行）。
2. Authentication → Settings → 授權網域加 `你的帳號.github.io` 與 `localhost`。
3. 部署安全規則（二選一）：
   - **CLI（推薦，自動化）**：複製 `安裝包/firestore.rules` → 根目錄 `./firestore.rules`、把 `STUDIO_EMAIL@gmail.com` 換成教室信箱 → `firebase deploy --only firestore:rules`。
     一次性前置：`npm i -g firebase-tools` → `firebase login`（瀏覽器授權）→ 手動建 `.firebaserc`、`firebase.json`（**別跑 `firebase init`**，會 scaffold 覆蓋）。詳見 `安裝指引.md` 段 1。
   - **Console（備援）**：Firestore Database 頁**上方分頁**「規則(Rules)」（不在左側選單）→ 貼 `安裝包/firestore.rules`、改信箱 → 發布。找不到分頁就用 CLI。
   > 安裝包唯讀：填 email 一律在根目錄工作副本 `./firestore.rules`，不要改 `安裝包/` 那份。

### 🔧 卡關速查（資料庫 / 規則）
- **agy 走 CLI 部署規則前，先裝工具**：`npm i -g firebase-tools` → `firebase login`（瀏覽器授權，這步人工，agy 停下等老師）。沒裝就部署會失敗。
- **Console 顯示「建立資料庫」≠ 沒建**：若連線測試已過，資料庫一定存在（Firestore 無法對不存在的庫讀寫）。多半是頁面沒跟上 → **強制重新整理**，或開 `…/firestore/databases/-default-/data`。**別亂按「建立資料庫」**（可能建出第二個空庫）。
- **規則分頁在哪**：Firestore Database 進到資料庫後，**主畫面上方**那排「資料 / 規則 / 索引」，不在左側選單。
- **確認資料庫存在與地區**：`firebase firestore:databases:list` → 看有沒有 `(default)`、`locationId` 是不是 `asia-east1`。若地區不對（CLI 預設常落美國），功能仍可用但延遲較高，要換區得重建空庫再重設。
