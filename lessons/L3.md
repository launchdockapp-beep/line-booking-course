### 🧰 設定關卡 1（老師，課程開始）— 建 LINE Login＋LIFF
> 鐵律：在**你 OA 所在的同一個 Provider** 下建，userId 才會與推播一致。
1. developers.line.biz → 進 OA 的 Provider → Create channel → **LINE Login**。
2. 該 channel → LIFF → Add：Endpoint URL = 你的 Pages 網址、Size Full、scope 勾 profile+openid → 取得 **LIFF ID**。

### 👩‍💻 給 agy 的指令（這一課）
```
只做 L3：載入官方 LIFF v2 SDK（https://static.line-scdn.net/liff/edge/2/sdk.js）。
liff-init.js：liff.init({liffId:常數})→未登入則 liff.login()→getProfile 取 userId/displayName 存全域。
查 members/{userId}：有→頂端顯示「{name},剩{credits}堂,{expiresAt YYYY/MM/DD}到期」；
無→一次性註冊表單(姓名,電話)→建 members/{userId}{displayName,name,phone,credits:0,status:"pending",createdAt}（符合 firestore.rules 會員自建規則）。
LIFF_ID 留常數待我填。
**不要用 setTimeout 等匿名登入/db 就緒**；改用 firebase.auth().onAuthStateChanged() 或 await 匿名登入 Promise 後再查 Firestore（事件驅動，不要猜延遲）。
做完停下。不要做預約。
```
